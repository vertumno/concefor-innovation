## Como esta pasta funciona

Esta pasta é organizada por **engenharia de contexto**: separa o que o time *sabe* (contexto)
do que o time *faz* (produção) e de como o time *se organiza* (planejamento). Qualquer pessoa —
ou IA — que precise agir deve conseguir carregar o contexto e entender o estado de tudo.

```
comunicacao/
├── _inbox/            ← Jogue aqui tudo que surgir. Depois processamos e distribuímos.
├── contexto/          ← O CÉREBRO. Fatos canônicos do evento (fonte de verdade).
├── producao/          ← As peças sendo produzidas, por fase (brindes, pré, durante, ideias).
├── planejamento/      ← Board de produção, calendário de publicação, distribuição da equipe.
├── templates/         ← Modelos de brief e roteiro para iniciar uma peça nova.
├── site-concefor.md   ← Cópia bruta do conteúdo do site oficial (fonte primária).
└── README.md          ← Você está aqui.
```

### O fluxo do `_inbox`
1. Surgiu qualquer coisa (ideia, arquivo, link, pedido)? → jogue em [_inbox/](_inbox/)
2. Em algum momento processamos o inbox: cada item vira uma peça em `producao/`,
   um fato em `contexto/`, ou uma tarefa no board de `planejamento/`.
3. Item processado sai do inbox.
